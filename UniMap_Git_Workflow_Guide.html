0) Ce ai deja

✅ Repo GitHub: cosmintrica/UniMap   •   ✅ SSH configurat • ✅ primul push reușit.

De aici folosim doar comenzi simple. Fiecare pas are copy‑button.

1) Concept — cum lucrăm

  • main = ramura stabilă, mereu funcțională.
  • Pentru orice schimbare: branch nou → commit → Pull Request → merge în main.

2) Începe un feature (branch nou)

Din folderul proiectului (acolo unde e UniMap.xcodeproj):

git switch main
git pull

# creează și treci pe un branch nou
git switch -c feat/onboarding-polish

Lucrezi în Xcode. Când vrei să salvezi schimbările:

git add -A
git commit -m "polish: onboarding UI + compact reset panel"

Trimite branch-ul pe GitHub:

git push -u origin feat/onboarding-polish
Pe GitHub apare butonul Compare & pull request. Apasă-l pentru a deschide PR-ul.

3) Pull Request & merge în main

  1. Completezi titlu + descriere în PR.
  2. Verifici că trec verificările (dacă ai workflow).
  3. Merge (recomandat: Squash and merge).
  4. Ștergi branch‑ul din GitHub (butonul Delete branch).

Apoi sincronizezi local:

git switch main
git pull
git branch -d feat/onboarding-polish

4) Protejează ramura main

GitHub → Repository → Settings → Branches → Add branch protection rule:

  • Branch name pattern: main
  • Bifează: Require a pull request before merging
  • (Opțional) Require status checks to pass — dacă ai workflow de build.

5) Invită colaboratori

Repo → Settings → Collaborators → Add people → introdu username‑ul → alege permisiunea.

6) README & CONTRIBUTING

Adaugă fișierele de bază:

cat > README.md << 'EOF'
# UniMap

Indoor maps pentru universități (Craiova → prima).
**Stack:** SwiftUI, MapKit, JSON pentru hărți, editor intern.

## Rulare
1. Deschide `UniMap.xcodeproj`
2. Schema: UniMap
3. Rulează (⌘R)

## Structură
- App/ — entry
- Features/ — ecrane
- Shared/ — componente comune
- Data/Maps — hărți JSON

EOF

cat > CONTRIBUTING.md << 'EOF'
## Flux de lucru
1) `git switch main` + `git pull`
2) `git switch -c feat/nume-feature`
3) `git add -A` + `git commit -m "feat: ..."` + `git push -u origin feat/...`
4) Creează Pull Request → „Squash and merge”
5) `git switch main` + `git pull` + `git branch -d feat/...`
EOF

git add README.md CONTRIBUTING.md
git commit -m "docs: add README & CONTRIBUTING"
git push

7) Git LFS pentru imagini mari

brew install git-lfs        # o singură dată
git lfs install
git lfs track "*.png" "*.jpg" "*.jpeg" "*.pdf"
git add .gitattributes
git commit -m "chore: enable Git LFS"
git push
După asta, poți urca screenshot-uri mari fără probleme.

8) Build automat (GitHub Actions)

mkdir -p .github/workflows
cat > .github/workflows/ios-build.yml << 'EOF'
name: iOS Build

on:
  pull_request:
    branches: [ main ]
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: macos-14
    steps:
      - uses: actions/checkout@v4
      - name: Select Xcode
        run: sudo xcode-select -s /Applications/Xcode.app
      - name: Resolve SPM
        run: xcodebuild -resolvePackageDependencies -project UniMap.xcodeproj
      - name: Build (Debug)
        run: xcodebuild -project UniMap.xcodeproj -scheme UniMap -destination 'platform=iOS Simulator,name=iPhone 15' build | xcpretty
        shell: bash
EOF

git add .github/workflows/ios-build.yml
git commit -m "ci: add iOS build workflow"
git push

9) Probleme frecvente

„Updates were rejected…” (remote are schimbări):

git pull --rebase
git push

Nu mă lasă să schimb branch-ul (ai modificări necomise):

# salvezi temporar
git stash
git switch main
git stash pop

# sau faci un commit provizoriu
git add -A
git commit -m "WIP"

Remote pe HTTPS (cere user/parolă) — treci pe SSH:

git remote set-url origin git@github.com:USERNAME/UniMap.git
git remote -v

10) Xcode GUI (alternativ)

  • Init/Commit/Push: Source Control → Commit… → Push…
  • Branch nou: Source Control → New Branch…
  • Schimbă branch: Source Control → Switch to Branch…
  • PR: după push, mergi pe GitHub → Compare & pull request.
